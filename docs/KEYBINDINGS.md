## How Keybindings Work

Keybindings are declared as `(key, command, mode)` tuples in each layer's
`_keybindings()` method. During apply, the `LayerStack` merges them by
priority — higher-priority layers win on key conflicts.

```
BaseLayer[10] → BehaviorLayer[40] → PrivacyLayer[20] → UserLayer[90]
                                                          ↑ always wins
```

To override any binding, add it to `layers/user.py`.
To see all bindings in the running browser: `:bind`

## Customising Keybindings

Add to `layers/user.py`:

```python
def _keybindings(self) -> List[Tuple[str, str, str]]:
    L = self._leader
    return [
        # Open clipboard URL in new tab
        ("gx", "open -t -- {clipboard}", "normal"),
        # Copy as Markdown link
        (f"{L}M", "yank inline [{title}]({url})", "normal"),
        # Quick search selection
        ("ss", "hint --rapid links tab-bg", "normal"),
    ]
```

Changes take effect on `,r` (`:config-source`).

## Conflict Detection

Run from the project root:

```bash
python3 -c "
from keybindings.catalog import KeybindingCatalog
from layers.base import BaseLayer
from layers.behavior import BehaviorLayer
from layers.privacy import PrivacyLayer, PrivacyProfile
from layers.user import UserLayer

catalog = KeybindingCatalog.from_layers([
    BaseLayer(), BehaviorLayer(), PrivacyLayer(), UserLayer()
])
print(catalog.conflict_report())
"
```
